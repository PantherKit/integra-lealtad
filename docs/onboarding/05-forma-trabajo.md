# Forma de Trabajo — Coutiño x Integra Group AI

**Proyecto:** Lealtad Digital

---

## 1. Metodología

Trabajamos con entregas incrementales y pautadas. Esto significa que no desaparecemos 8-10 semanas y entregamos todo al final. En su lugar:

- Cada semana hay un entregable visible
- Cada entregable se revisa y aprueba antes de avanzar
- Si algo no va bien, lo detectamos temprano y corregimos

---

## 2. Ciclo Semanal

| Día | Actividad |
|---|---|
| Lunes | Planeación semanal — definimos qué se entrega esta semana |
| Martes - Jueves | Desarrollo y avance |
| Viernes | Demo / Revisión — mostramos lo que se hizo, recibimos feedback |

**Reuniones fijas:**

| Reunión | Frecuencia | Duración | Participantes |
|---|---|---|---|
| Kick-off | Una vez (inicio) | 60 min | Equipo completo ambas partes |

---

## 4. Proceso de Aprobación

Cada entregable sigue este flujo:

1. **Integra Group AI entrega** → Notificamos por email/WhatsApp con link al entregable
2. **Coutiño revisa** → Tienen 3 días hábiles para revisar y dar feedback
3. **Feedback** → Si hay cambios, los incorporamos en la siguiente iteración
4. **Aprobación** → Confirmación por email o WhatsApp de que el entregable es aceptado
5. **Siguiente fase** → Avanzamos al siguiente bloque

Si no recibimos feedback en 3 días hábiles, el entregable se considera aprobado y avanzamos.

---

## 5. Gestión de Cambios

| Tipo de cambio | Cómo se maneja |
|---|---|
| Ajuste menor (UI, textos, configuración) | Se incorpora sin costo adicional |
| Funcionalidad nueva no contemplada | Se evalúa, cotiza y documenta como adenda |
| Cambio en reglas de negocio | Se evalúa impacto y se ajusta timeline si es necesario |
| Cambio de alcance significativo | Requiere aprobación escrita y ajuste al contrato |

---

## 6. Escalamiento

Si hay un problema o bloqueo:

| Nivel | Situación | Acción |
|---|---|---|
| 1 | Duda o ajuste menor | WhatsApp con punto de contacto |
| 2 | Bloqueo técnico o de información | Email + reunión extraordinaria |
| 3 | Desacuerdo en alcance o dirección | Reunión con decisores de ambas partes |

## 7. Entregables por Fase

| Fase | Entregable | Formato |
|---|---|---|
| Discovery | Documento de requerimientos aprobado + wireframes (incluye diseño Wallet pass) | Google Doc + Figma |
| Diseño técnico | Arquitectura técnica + modelo de datos + flujo Wallet | Doc + diagrama |
| Desarrollo Sprint 1 (Backend) | Backend funcional + módulos base + tests | Demo en vivo |
| Desarrollo Sprint 2 (Frontend) | Frontend (editor + dashboards) integrado | Demo en vivo |
| Desarrollo Sprint 3 (Motor) | Motor de puntos + flujo cliente final E2E | Demo en vivo |
| Integraciones core | Stripe + WhatsApp + SES funcionando | Demo en vivo |
| Apple Wallet + Google Wallet | Pass operativo en iOS real + Loyalty Object en Android real | Demo en vivo (con teléfono físico) |
| Piloto & Testing | Reporte de pruebas + bugs resueltos + validación con 2-3 comercios | Doc |
| Entrega | Sistema en producción + documentación + workshop | Sesión en vivo |
